Progress Update

April 8–10, 2026

WIN — APRIL 8

Anyone on the team can now generate a professional daily report in under a minute — no technical skills, no AI, just tap and copy.

WIN — APRIL 9

The app now has a project board where completed tasks automatically show up in the daily report — progress tracks itself.

WIN — APRIL 10

PickD can now spot when inventory has been silently inflated and roll it back to the truth — and starting tonight, daily reports build themselves at midnight from real activity (orders completed, items moved, corrections made, project tasks closed), giving leadership a clear picture of the day even when no one writes a single line.

WHAT'S BETTER NOW

●  A hidden flaw was secretly adding phantom units to inventory every time an old order got auto-cancelled. The system now catches this kind of drift and rolls counts back to what's actually on the floor.
●  Daily reports now save themselves at midnight as locked snapshots. Yesterday's numbers can never be accidentally rewritten by someone editing today's report.
●  There's a new Activity Report screen where you fill in a few quick fields, hit Copy, and paste a clean summary into email. Order counts and team activity fill in on their own.
●  A new Projects board lets you move tasks between “Coming Up”, “In Progress”, and “Done”. Whatever gets marked done feeds straight into the daily report.
●  Inventory accuracy used to count only items that were manually audited — less than 1% of stock. Now the system also counts items that were physically moved or received, bringing coverage above 20% with zero extra work.
●  Searching for an item that's out of stock no longer shows a confusing full card. Instead, a small “No stock” section tells you what happened — moved, shipped, or deleted — and when.
●  The activity report used to show today's project board even when looking at past days. Now it shows what was actually true on that day — historical reports finally tell the real story.
●  Every bike now gets a printable label with a QR code. Scanning it opens a clean public page with the bike's photo, model, and basic info — no app or login needed.
●  During verification, the checker can now scan a pallet's QR code and snap a proof photo. The photo is attached to the order as physical evidence of what shipped.
●  Cycle count used to only let you find SKUs that were already on screen. Now you can search the whole warehouse and add any item on the spot.
●  Editing an item from the verification screen used to make the whole edit window vanish if you closed the picking drawer behind it. The edit window now stays open until you finish.
●  Action colors across the app used to be inconsistent — the same red could mean “delete” in one screen and “warning” in another. Every color now has a single meaning, making the app easier to read at a glance.
●  The profile menu used to get stuck open and block other screens. It now closes cleanly so you can keep moving.
●  The Picking Summary used to show one giant card per item that took up half the screen. It now fits the whole order in a fraction of the space, with item names visible at a glance and any correction notes one tap away.
●  Cleaned out 76 leftover test records (orders, products, and notes) that were polluting production data. Two of those notes accidentally contained sensitive setup info from months ago — that has been removed and the affected key was rotated.
●  Closed a serious security gap where any team member could have technically promoted themselves to admin. The system now blocks role changes from non-admins at the database level, no matter how the request comes in.
●  The main menu was restructured: warehouse activities are now front and center, profile and settings live in a clean sub-panel, and a bunch of dead code was removed along the way.
●  Customer addresses with numbered streets like “123 4th N” used to break the auto-fill when pasted from a multiline format — the parser now handles them correctly.
●  All dates and times across the app now follow a single source of truth for New York time. Past edge cases around midnight (where the same day showed different totals depending on who was looking) are gone.
●  The picking drawer used to stay open in the background when you navigated away from picking. It now closes cleanly so it doesn't pop up unexpectedly on other screens.

🔒 SECURITY HARDENING

A full security audit was performed across 7 fronts. Critical issues were identified and the safest fixes were applied to production today.

✓  Privilege escalation closed: a database trigger now blocks any non-admin from changing user roles, even by going around the app.
✓  Project board permissions were claiming to be admin-only but actually let anyone write — fixed.
✓  13 internal database functions hardened against a known attack pattern. No behavior change, just defense in depth.
✓  Test admin password was leaking into the public web bundle — removed.
✓  A daily report generator that was silently crashing every night has been fixed.

Higher-impact fixes (broader permission rewrites, server endpoint authentication) are scheduled for the next phase to avoid disrupting active workflows.
