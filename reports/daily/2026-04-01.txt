Progress Update — April 1, 2026 at 4:28 PM

WIN OF THE DAY:
Found and fixed the root cause of orders disappearing during verification — the system now protects active sessions from being overwritten, and grouped orders release properly when stepping back.

WHAT'S BETTER NOW:
- There are now two versions of the app: the stable one the team uses every day, and a preview version where new features get tested first. If something breaks in preview, the real app stays untouched.
- The preview version shows a yellow "STAGING" banner at the top so nobody confuses it with the real thing.
- Searching for a location with many items (like ROW 52 with 23 bikes) used to cut off results — now it shows all of them and loads more automatically as you scroll.
- When a picker is working on an order and another verification is in progress, the system no longer loses track of the order being edited. This was the bug that hit Jed today.
- Grouped orders (like FedEx shipments with multiple sub-orders) now release all siblings correctly when stepping back, instead of leaving some locked.
- The system was keeping a growing pile of internal housekeeping records that took up half the storage — cleaned it up and set it to auto-clean every week from now on.

ON THE FLOOR:
