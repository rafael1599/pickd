Progress Update

April 1, 2026
 

🏆 WIN OF THE DAY

Found and fixed the root cause of orders disappearing during verification — the system now protects active sessions from being overwritten, and grouped orders release properly when stepping back.

✅ WHAT'S BETTER NOW
🔒 Stable + Preview versions — There are now two versions of the app: the stable one the team uses every day, and a preview version where new features get tested first. If something breaks in preview, the real app stays untouched.
🚨 Staging banner — The preview version shows a yellow "STAGING" banner at the top so nobody confuses it with the real thing.
📋 Full search results for large locations — Searching for a location with many items (like ROW 52 with 23 bikes) used to cut off results. Now it shows all of them and loads more automatically as you scroll.
🛠 Verification session protection — When a picker is working on an order and another verification is in progress, the system no longer loses track of the order being edited. This was the bug that hit Jed today.
🔗 Grouped orders release fix — Grouped orders (like FedEx shipments with multiple sub-orders) now release all siblings correctly when stepping back, instead of leaving some locked.
🗑 Storage cleanup + auto-clean — The system was keeping a growing pile of internal housekeeping records that took up half the storage. Cleaned it up and set it to auto-clean every week from now on.

📦 ON THE FLOOR
🚲 New container unloaded — Registered 21 bikes into ROW 52: Faultline A1/A2/A3, Renegade S1 UDH Framekits, Renegade C4, and Renegade S1 Adobe Clay. All logged with names, dimensions, and pallet distribution.

🔄 TRIED BUT NEEDS REWORK
🛠 Inline correction for verification — Built a first version where the checker could see similar SKUs and swap items directly in the double-check screen. After testing, the approach didn’t work well: quantities were confusing, suggestions weren’t useful, and when no alternatives exist the button shouldn’t appear at all. Next step: redesign as a dedicated “Correction Mode” screen instead of inline controls.

⏭ COMING UP NEXT
🔄 Correction Mode: a dedicated screen for fixing problem items during verification — search freely for replacements from the full inventory, not just similar SKUs.
👁 Redesign how the system handles unfinished orders: keep them visible instead of silently canceling them.
