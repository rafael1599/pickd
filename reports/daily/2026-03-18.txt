Progress Update — March 18, 2026


Summary:

Set up watchdog-pickd as a persistent service on the Bay 2 MacBook, deployed the auto-combine orders feature to production, added total weight display to shipping labels with missing-weight warnings, finalized correct weights for all bike categories, and improved the distribution editor with smart defaults and UX fixes.


--- Details below ---


Bay 2 Setup — Persistent Service:

- Cloned the watchdog-pickd repository fresh on the Bay 2 MacBook and configured the environment (venv, dependencies, .env credentials).
- Ran the watcher for the first time, which automatically installed a launchd service (com.antigravity.watchdog-pickd). This means the PDF watcher now starts automatically on login and restarts itself if it ever crashes — no need to keep a terminal open.
- Tested and confirmed the service survives machine restarts. Orders can continue to be sent from Bay 2 without any manual intervention.


Auto-Combine Orders — Deployed:

- Applied the database migration (combine_meta column) to production via supabase db push.
- Pulled the latest watchdog-pickd code on Bay 2 with the combine logic active. From now on, when multiple PDFs come in for the same customer, the system will automatically merge them into a single order.
- Pending: test with real orders — need different order numbers arriving for the same customer to confirm the combine works end-to-end in production.


3D Printing — Location Numbers:

- All numbers (0–9) are now fully printed and ready to go. Next step is to start labeling the warehouse aisles/islands with them.


Shipping Labels — Weight Display:

- Shipping labels now show "WEIGHT: X LBS" after the LOAD field, both in the on-screen preview and the printed PDF.
- The weight is calculated automatically from each SKU's weight_lbs in the metadata, multiplied by the quantity picked. Different SKUs with different weights are handled individually.
- All bike categories now have their correct weights set: kids bikes (35 lbs), adult bikes (45 lbs), e-bikes (60 lbs), and specific models like 07-3686BK (28.5 lbs).
- Going forward, every new item added to inventory must be entered with its correct weight. This prevents inconsistencies when generating the BOL and avoids correction claims from carriers.


Missing Weight Warning — Orders View:

- When an order contains items that have no weight assigned in sku_metadata, a yellow warning banner now appears above the shipping label preview.
- The banner lists each SKU missing weight with an inline input field. The user types the weight in lbs and presses Enter to save it directly to sku_metadata.
- Once saved, the total weight on the label updates immediately — no need to leave the screen or go to inventory to fix it.
- This prevents shipping labels from being printed with inaccurate "N/A" or zero weight.

Distribution Editor — Smart Defaults:

- New distribution rows now copy the type from the previous row (instead of always defaulting to TOWER). First row defaults to LINE.
- units_each auto-fills with the type's standard amount (LINE=5, TOWER=30, PALLET=10) but caps at remaining units to avoid over-distributing. Example: 21 units with 20 already distributed → new row gets units_each=1.
- Number fields (count, units_each) now allow clearing to type a new value. Previously the minimum-1 constraint blocked deleting the last digit.
- Weight field in Edit Item now rejects negative values (schema validation). The Update button also correctly detects weight and dimension changes.


Realtime Sync Improvement:

- Added realtime sync for sku_metadata changes (dimensions + weight). If someone updates these fields on another device, the open edit form now reflects the change immediately. Previously, only distribution had live sync — now all metadata fields do too.
