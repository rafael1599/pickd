# PDF pipeline ‚Äî how the report renders (+ gotchas)

The report deliberately matches PickD's **Activity Report** PDF
(`src/features/reports/pdf/ActivityReportPdfDoc.tsx`) so it looks like a
first-party PickD artifact. Same engine, tokens, and fonts.

## Engine

- **`@react-pdf/renderer`** (installed in pickd, v4.x) ‚Äî vector output with
  selectable text. We use `Document/Page/View/Text/Image/Svg/Path/Circle` and
  `renderToFile`.
- The template is **CommonJS + JSX**; `build.cjs` transforms the JSX with
  **esbuild** (a transitive pickd dep, classic runtime ‚Üí `React.createElement`)
  and renders it with Node. No global installs; `build.cjs` finds pickd's
  `node_modules` by walking up from its own location.

```bash
node build.cjs /tmp/this-week.jsx /tmp/weekly-report.pdf
```

## Visual language (tokens mirror src/lib/pdf/tokens.ts)

- Background `paperWarm #FAF8F5`; 5pt **teal `#0E8C6B`** top band; teal corner
  marks; eyebrow `PICKD ¬∑ WAREHOUSE OPERATIONS`; big title; right-aligned date.
- Numbered **teal pills** (01, 02‚Ä¶) + caps section label + hairline, per win.
- White cards (`paperPure`, `hair #E6E4DE` border) for figures.
- **"What you gain"** in `tealDeep`. Before/after uses `red #B42318` (with
  strikethrough for the "before") ‚Üí `teal` for the "after".
- Mono footer: `GENERATED BY PICKD ¬∑ <date>` + `PAGE x / N`.
- Fonts: **Inter** (400/500/600/700) + **JetBrains Mono** (400/600) from
  `@fontsource`, registered from the `.woff` files in `node_modules`.

## Figure library (in report-template.jsx)

Reusable, faithful **vector mockups** in the app palette ‚Äî they read sharper than
a cramped screenshot and never need live data:

- `MiniOrder` ‚Äî order card (before/after customer, dashed-red error state).
- `Screen` ‚Äî tiny app window (blank/error vs working).
- `Tag` + `Arrow` + `Check` ‚Äî chips and a translate/flow row (e.g. SKU flow).
- `LabelMock` + `QR` + `Barcode` ‚Äî a print label (dense vs clean).
- `TCell` table ‚Äî the AS400 mini-table (MOVED FROM / CURRENT STOCK / TOTAL).
- `ScoreCard` / `RecapBand` ‚Äî the Before‚ÜíAfter scoreboard.
- `ImageFig src="‚Ä¶"` ‚Äî **embed a real screenshot** if the user provides a PNG.

Compose these per report; add new ones from the same atoms. Keep each figure ~75‚Äì
120pt tall so a section is one clean row.

## ‚öÝ Gotchas (each one cost a debugging cycle)

1. **The woff subsets lack `‚Üí`, `‚úì`, and emoji (`üîé` ‚Ä¶).** Typing them renders a
   tofu box, a stray `'`, or a silently-dropped glyph. **Draw them as SVG** ‚Äî use
   the provided `Arrow`, `Check`, `Mag`. Never put these glyphs in a string. `¬∑`
   `¬´¬ª` `‚Äî` `‚Ä¶` DO render and are fine.
2. **No italic font is registered.** `fontStyle: 'italic'` throws
   _"Could not resolve font ‚Ä¶ fontStyle italic"_ and kills the render. Convey
   emphasis with color/weight instead, never italic.
3. **JS string quotes vs apostrophes.** Body copy with an apostrophe (`won't`,
   `what's`, `order's`) inside a **single-quoted** JS string terminates the string
   ‚Üí esbuild "Expected }". Use **double quotes** for any string containing `'`.
4. **`renderToFile` is async.** The template exports `async (out) => ‚Ä¶`; `build.cjs`
   awaits it. Don't assume the file exists synchronously.
5. **Pagination.** Each `Section` is `wrap={false}` so it never splits. If a page
   ends with a big gap, either let content reflow (remove forced breaks) or add the
   `RecapBand` to fill an underfull page 2. Avoid orphaned headers.

## Preview before sending (mandatory)

Render every page to PNG and look at it ‚Äî don't trust the build succeeding.

```python
import fitz  # PyMuPDF
d = fitz.open('/tmp/weekly-report.pdf')
for i in range(d.page_count):
    d[i].get_pixmap(matrix=fitz.Matrix(2.3, 2.3)).save(f'/tmp/wr_p{i+1}.png')
```

(or `pdftoppm -png -r 150 /tmp/weekly-report.pdf /tmp/wr`). Zoom into any figure
with small text/icons (use a `clip` rect) to confirm arrows, checks and the
strikethrough render. Check: no tofu, balanced pages, nothing clipped, the date
and "for <name>" are right. Only then `SendUserFile`.
